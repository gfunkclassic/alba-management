# AI Development Rules

이 프로젝트에서 AI(Claude, Gemini, Copilot)는 다음 규칙을 반드시 따라야 합니다.

---

# 1. 수정 범위 제한

요청된 기능만 수정합니다.

기존에 문제 없이 동작하던 코드는 절대 수정하지 않습니다.

---

# 2. 최소 변경 원칙

코드 수정은 반드시 **최소 변경(minimal diff)** 으로 진행합니다.

불필요한 변경 금지

- 코드 포맷 변경
- import 정리
- 변수 이름 변경
- 파일 이동
- 리팩터링

---

# 3. 변경 전 확인

코드 수정 전 반드시 확인

- 수정 대상 파일
- 수정 대상 함수
- 수정 영향 범위

---

# 4. 변경 후 보고

코드 수정 후 반드시 보고

- 수정한 파일
- 수정한 함수
- 변경된 코드 설명
- 기존 기능 영향 여부

---

# 5. CI 통과 필수

모든 변경은 Pull Request를 통해 진행합니다.

다음 CI가 통과해야 merge 가능합니다.

CI / Web build (Vite)

CI / Functions build (TypeScript)

---

# 핵심 원칙

요청된 코드만 최소한으로 수정하고  
기존 기능은 절대 변경하지 않는다.
